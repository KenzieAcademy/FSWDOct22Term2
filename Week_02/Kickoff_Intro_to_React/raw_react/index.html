Vocabulary

  • Library - a set of pre-defined functions, classes, data types, etc., that you can borrow and use in your own application.
  • React - (also known as React.js) is a JavaScript library that allows us to create reusable "components" to build comprehensive client-side web single page applications
  • Single Page Application - a client-side application that technically only uses a single HTML page. The different pages that a user interacts with are chunks of DOM elements swapped in and out by JavaScript
  • Framework - similar to a library in that you are using resources created by someone else, but where it differs it that your code must follow the flow of the framework.
  • JSX - JSX is a hybrid of JavaScript and HTML that allows you to used JavaScript logic mixed between HTML code.
  • Babel - a JSX transpiler that converts JSX code into browser-readable JavaScript
  • Component - In React at least, a component is a function (or class, but we'll talk more later on that) that returns JSX

    • What is React?

    React is a JavaScript library that is used to build (or enhance) client-side (a.k.a. Front End) web applications.
